Optional clientName

clientName: string
  • Defined in index.d.ts:1678

所代表的的客户端名称 当我们有多个答题客户端时，在设计端发布问卷之前，可以选择使用哪个答题端来渲染问卷。 一般选好以后，按需求这份问卷就只能使用这个客户端来渲染，因为如果不限制仅能在该客户端渲染， 可能会导致不知不觉中误用其他客户端来访问这份文件的情况， 这样就会： 一方面：不是所有的客户端都实现了所有题型，用错客户端可能导致有些题目无法渲染。 另一方面，不同客户端对相同题型渲染方式不同，用错客户端可能导致渲染出来的样子不是客户需要的样子。

但事实上默认情况下，是所有客户端都能访问该问卷的

为了避免上面的两个错误出现，我们可以通过这个属性为每个客户端设置一个唯一的名称。

访问某份问卷前核心包会检查当前客户端名字是否和该问卷在设计端选择的客户端名字相同，只有相同才能访问。

如果名称为ALL则表示这个客户端能访问所有问卷

默认值：ALL

Optional langSrcMap

langSrcMap: object
  • Defined in index.d.ts:1821

如果觉得核心包中逻辑相关的提示文字多语言翻译不满足你的要求，或者支持的语言不够多 你可以配置该属性指定使用自己准备的翻译文本，这个配置的每个属性的属性名是语言代号， 其属性值是该语言翻译文本（json格式）所在的网络地址 比如，核心模块当前仅提供了中文（zh_cn）和英文（en_us）的多语言文本 现在你对核心模块的中文翻译是满意的但是对核心模块的英文的翻译内容不满意， 这样你就可以配置该属性如下内容

(我们假设你将多语言文本放到了下面这个网址中)

{
 "en_us": 'https://yousite.com/your-os-client/es_us.json'
}

以上配置会覆盖掉默认的英文翻译，当进行英文翻译时，会只用你提供的这本json作为翻译依据， 核心模块提供的英文翻译会被丢弃

如果在此基础上,同时你希望有日文版的多语言翻译，这时你可以添加配置日文翻译文本

{
 "en_us": 'https://yousite.com/your-os-client/es_us.json'
 "ja_jp": 'https://yousite.com/your-os-client/ja_jp.json'
}

以上的配置覆盖了英文翻译，同时新增了日文翻译。

每本json要准备的数据格式在LANG_ZH_CN和LANG_EN_US中做出了示范.

各种语言的代号可参照百科全书

我们统一使用xx_xx小写字母加下划线形式的代号

默认值：空

Optional setter

setter: CFPropSetFn
  • Defined in index.d.ts:1703
deprecated

该属性仅为兼容老版UI中使用的emberjs的setProperties暂时保留，即将移除，不要使用这个属性

该属性已废除，请使用最新标准：使用CFEventHub来监听SET_PROPS来监听变化重新更新UI 属性设置辅助方法 由于UI中绑定了核心提供的数据结构的某些值，UI中进行操作后回调了核心提供的回调函数 核心中的逻辑会更改数据结构中的这些值，核心逻辑中默认是直接更改这些值的。 这样的话，大部分框架的UI模板引擎是无法觉察到这个更改的，从而UI上的内容不会正常更新。 这时候可以通过设置该方法来使得模板引擎觉察到内容的更改达到更新页面的目的。

举例：UI中渲染选项时背景颜色绑定了CFOption.selected属性， 选中的选项（option.selected为true）渲染成红色， 未选中的选项（option.selected为false）渲染成灰色， 现在某个选项初始时是未选中的，根据绑定规则，渲染成了灰色，现在点击该选项，会回调核心 逻辑中提供的选项点击回调函数，在该函数中会将option.selected改为true， 但是UI模板并不知道这个属性发生了变化，所以依然渲染成了灰色，这时就需要用的这个配置方法。 如使用emberjs的时候我们将该方法配置为EmberObject的setProperties方法,如果不配置则 UI不会得到更新.

  import {setProperties} from '@ember/object';
  options.setter = setProperties

默认行为：只更新数据，不做其他任何行为。

Optional switchRoute

  • switchRoute(routeParam: CFRouteSwitchParam): void
    • Defined in index.d.ts:1851

    自定义的去往开始页面方法,只有使用web核心包是需要关注这个方法

    非web核心包使用者无需配置该方法。

    web定制核心包中配置的页面切换方法是使用的浏览器原始location.replace方法实现的 当使用了单页引用页面的框架实现web客户端时，原生的跳转方法可能会导致页面刷新，使得单页应用的状态丢失。 而不同的框架在单页面内控制路由控制的方式是不一样的，所以需要用该方法提供。
